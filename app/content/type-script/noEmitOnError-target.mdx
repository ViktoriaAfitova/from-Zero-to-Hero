---
title: Compile options noEmitOnError & target
---

Параметры компиляции **noEmitOnError** и **target** в TypeScript позволяют управлять процессом компиляции и целевой версией JavaScript кода.

#### noEmitOnError:

- Описание: Этот параметр указывает компилятору TypeScript не генерировать выходной JavaScript код, если в процессе компиляции возникают ошибки.
- Значение по умолчанию: false (выходной код будет сгенерирован, даже если есть ошибки).
- Пример использования в tsconfig.json:
```json
{
  "compilerOptions": {
    "noEmitOnError": true
  }
}
```
Если установлено значение true, компилятор TypeScript не сгенерирует JavaScript код (.js файлы) при наличии ошибок в исходных файлах TypeScript (.ts файлы). Это может быть полезно, чтобы предотвратить генерацию неполноценного или ошибочного кода при наличии проблем.

#### target:

- Описание: Этот параметр указывает, какая версия JavaScript будет целевой для компиляции.
- Значение по умолчанию: **ES3**.
Допустимые значения: **ES3**, **ES5**, **ES6** / **ES2015**, **ES2016**, **ES2017**, **ES2018**, **ES2019**, **ES2020**, **ES2021**, **ESNext**.
-Пример использования в tsconfig.json:
```json
{
  "compilerOptions": {
    "target": "ES2019"
  }
}
```
Параметр target определяет, какие возможности JavaScript будут использованы в сгенерированном коде. Более поздние версии target предоставляют доступ к новым возможностям языка JavaScript. Но, выбор более поздней версии target может ограничить поддержку старых браузеров, которые не поддерживают все новые возможности.

Регулярное использование параметров компиляции **noEmitOnError** и **target** позволяет контролировать и настраивать процесс компиляции TypeScript, чтобы обеспечить безошибочное исходное кодирование и соответствие целевой среде выполнения.
