---
title: memo, useMemo, useCallback
---

Хок memo и хуки useMemo, useCallback помогают оптимизировать производительность компонентов.

#### memo
**memo –** HOC(Higher Order Component) функция высшего порядка, которая используется для оптимизации функциональных компонентов. Она предотвращает ненужный повторный рендеринг компонента, если его входные свойства (props) не изменились. Это особенно полезно, когда компонент может рендериться чаще, чем это необходимо.

```jsx
import { memo } from 'react';

const MyComponent = memo((props) => {
  // Компонентная логика и отрисовка
});

export default MyComponent;
```

### useMemo
**useMemo –** хук, который позволяет кэшировать результат выполнения функции и повторно использовать его до тех пор, пока зависимости не изменятся. Это полезно, когда есть сложные вычисления, которые необходимо выполнять в компоненте.

```jsx
import { useMemo } from 'react';

const MyComponent = ({ items }) => {
  const expensiveCalculationResult = useMemo(() => {
    // Здесь происходят вычисления, использующие props.items
    return /* результат вычислений */;
  }, [items]); // Передаем зависимости в массиве

  // Остальная компонентная логика и отрисовка с использованием expensiveCalculationResult
};
```

#### useCallback
**useCallback –** хук, который возвращает мемоизированную версию функции, которая изменяется только тогда, когда изменяются её зависимости. Это полезно, когда требуеться предотвратить ненужное пересоздание функций в дочерних компонентах, которые передаются в качестве props.

```jsx
import { useCallback, useState } from 'react';

const MyComponent = () => {
  const [count, setCount] = useState(0);

  const increment = useCallback(() => {
    setCount((prevCount) => prevCount + 1);
  }, []); // Пустой массив зависимостей, т.к. здесь нет зависимостей

  // Остальная компонентная логика и отрисовка, использующая increment
};
```

Важно использовать **memo**, **useMemo**, и **useCallback** там, где это действительно нужно, чтобы предотвратить лишние вычисления и рендеры, и улучшить производительность вашего приложения.
