---
title: Mutable Vs Immutable
---

В React, мутабельность и иммутабельность касаются работы с данными, особенно с состоянием компонентов и пропсами.

**Мутабельность (Mutable)** относится к данным, которые могут быть изменены или модифицированы напрямую. Когда данные являются мутабельными, их значения могут быть изменены без создания новой копии объекта. Примером мутабельных данных в JavaScript являются объекты и массивы. Если изменения происходят напрямую в мутабельных данных, это может привести к неожиданным побочным эффектам и ошибкам, особенно при работе с React, где изменение состояния напрямую может нарушить логику перерисовки компонентов.

**Иммутабельность (Immutable)** предполагает, что данные не могут быть изменены напрямую после их создания. Вместо этого, при необходимости внести изменения, создается новая копия данных с внесенными изменениями. Таким образом, исходные данные остаются неизменными. Применение иммутабельности позволяет более безопасно и предсказуемо управлять данными, особенно в контексте работы с React, где это способствует более эффективному обнаружению изменений и перерисовке компонентов только при необходимости.

Иммутабельность в React особенно полезна для обновления состояния компонентов. Вместо изменения состояния напрямую, следует создавать новую копию состояния с помощью методов, таких как setState или хуков useState и useReducer. Такой подход позволяет React легко определить изменения в состоянии и эффективно перерисовывать только измененные части DOM.

Для работы с иммутабельными данными в JavaScript часто используются методы или библиотеки, которые предоставляют удобные способы создания копий данных с внесенными изменениями, например, Object.assign, оператор распространения (...), Array.map, Array.filter, а также библиотеки, такие как Immutable.js или immer.

Использование иммутабельности в React способствует созданию более стабильных, предсказуемых и производительных компонентов. Это также помогает предотвратить нежелательные эффекты, связанные с мутабельностью данных.
